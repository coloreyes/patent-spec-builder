# 专利说明书生成（主流程）

## 用途

将技术交底书或技术论文转换为标准**专利说明书**格式，输出两份文件：
1. **说明书摘要** — 一段式，200-400字
2. **说明书** — 完整格式，包含技术领域、背景技术、发明内容、附图说明、具体实施方式

## 输出格式要求

### 文件一：说明书摘要

**格式**：单一段落，不分节标题，约200-400字。

**内容结构**：
- 本发明公开了[方法/系统名称]，属于[技术领域]
- 针对现有[某技术]在[某场景]下面临的[问题1]以及[问题2]等缺陷
- 提出了[核心方案概述]
- 该方法首先[步骤1概述]，然后[步骤2概述]，进一步[步骤3概述]，最后[步骤4概述]
- 该方法在[某条件]下，能够[技术效果1]，提升[技术效果2]，具有[实际部署/应用价值]

### 文件二：说明书

**章节结构**（严格按以下顺序，无编号前缀）：

```
[发明名称]（首行，无标题标记，居中，黑体16pt加粗）

技术领域（黑体14pt加粗）
[1段]

背景技术（黑体14pt加粗）
[2-4段]

发明内容（黑体14pt加粗）
[目的段落]
[核心思路段落]
[步骤总述]
S1 [步骤名]：[详细描述]
S2 [步骤名]：[详细描述]
...
Sn [步骤名]：[详细描述]
[公式段落，OMath格式]
[系统模块段落]
[有益效果]（1）（2）（3）...

附图说明（黑体14pt加粗）
[图N为...说明]
[图片]
图N（居中，10.5pt）

具体实施方式（黑体14pt加粗）
[开头语段]
[术语解释段落（必选，6-10个术语）]
实施例1
[方法实施例，含详细公式推导和参数，每个步骤需展开3-5段]
实施例2
[系统实施例，模块对应关系]
[应用例（可选，含实验数据对比）]
[结语段]
```

### 撰写风格规范

1. **不使用**「一、二、三」或「第一章」等数字编号作为节标题
2. **节标题**仅用：`技术领域`、`背景技术`、`发明内容`、`附图说明`、`具体实施方式`
3. **步骤编号**使用 `S1`、`S2`...`Sn` 格式
4. **有益效果编号**使用 `（1）`、`（2）`... 格式
5. **公式**使用 Word 原生 OMath 格式（通过 `tools/omath_converter.py` 中的 `formula_to_omath(latex)` 函数转换后 XML 注入）
6. **表格**使用 Word 原生表格，标题用 Caption 样式
7. **术语**首次出现时给出中英文全称及简称，如「持续学习（Continual Learning, CL）」
8. **段落**首行缩进0.74cm，行距1.5倍，宋体12pt
9. **术语解释**在「具体实施方式」开头部分，以独立段落解释核心术语，格式为：
   `术语中文名（English Term）：通用定义。在本发明中，[该术语如何被使用或实现]。`
   - 必须包含 6-10 个术语，涵盖：核心技术概念、算法/机制名称、关键数学概念
   - 每个术语解释至少 50 字，分为「通用定义」+「在本发明中的应用」两部分
   - 示例格式：`协作关系图（Collaboration Graph）：指以智能体为节点、以智能体间交互关系为边的无向图结构。每条边包含关系类型和权重两个属性...在本发明中，协作关系图用于...`
10. **实施例1 展开**：每个步骤（S1-Sn）在实施例1中需展开为 3-5 段详细描述：
    - 第1段：该步骤的整体描述
    - 第2段：「具体的，设...」引入符号和公式推导
    - 第3-4段：公式的解释、参数说明、边界条件
    - 公式需完整展开（如编码器函数定义、映射层计算等）
11. **结语**以「本领域的普通技术人员将会意识到...」收尾

## 从交底书到说明书的映射规则

| 交底书章节 | 说明书章节 | 转换规则 |
|-----------|-----------|---------|
| 一、技术背景与现有技术 → 1.1 现有技术 | 背景技术 | 提炼技术脉络，去除检索说明和URL，改写为叙述性文字 |
| 一、1.2 现有技术缺点 | 背景技术（末段） | 融入背景技术，作为问题引出 |
| 二、本发明所要解决的技术问题 | 发明内容（首段） | 改写为「本发明的目的在于...」 |
| 三、3.1 背景 | 发明内容（核心思路） | 融合为核心思路段落 |
| 三、3.4 系统流程 S1-Sn | 发明内容（步骤） | 直接使用步骤描述，补充细节 |
| 三、3.4.1 符号与公式 | 发明内容/实施例 | 公式保留OMath格式，符号在实施例中展开 |
| 三、3.3 模块功能 | 发明内容（系统模块）+ 实施例2 | 模块描述拆分到发明内容（简述）和实施例2（详述） |
| 四、优点 | 发明内容（有益效果） | 改写为「与现有技术相比，本发明至少具有以下有益效果：（1）...」 |
| 五、技术关键点 | 融入全文 | 不再独立成节，内容融入各相关章节 |
| 六、实施例 | 具体实施方式 | 扩展为实施例1（方法）+ 实施例2（系统） |
| 3.2 系统框图/3.4 流程图 | 附图说明 | 生成图N说明并嵌入PNG图片 |
| 论文/交底书中的专业术语 | 术语解释 | 提取6-10个核心术语，按「中文名（英文）：通用定义 + 在本发明中的应用」格式撰写 |

## Word 生成技术要求

- 使用 `python-docx` 生成
- 公式使用 **OMath XML 注入**（Word 原生公式格式）：使用 `tools/omath_converter.py` 中的 `formula_to_omath(latex)` 函数将 LaTeX 公式转为 OMath XML 元素，通过 `paragraph._p.append(omath_element)` 插入段落
- OMath 命名空间：`http://schemas.openxmlformats.org/officeDocument/2006/math`
- 公式编号写在公式右侧：`（1）`、`（2）`...
- 表格使用 `document.add_table()`，标题用 `paragraph.style = 'Caption'`
- 正文段落使用 `style='Normal'`，首行缩进 `0.74cm`，行距 1.5 倍
- 节标题使用黑体 14pt 加粗，正文使用宋体 12pt

## 转换流程

1. **Read** 交底书全文（.md 或 .docx）
2. **分析**交底书结构，提取各章节内容
3. **重组**按照上述映射规则
4. **扩写**具体实施方式（补充详细公式推导、参数设置、术语解释）
5. **生成**说明书摘要（单段）
6. **输出**两份 Word 文档

## 输出文件命名

```
{案件名}_{YYYYMMDDHHmmss}-说明书摘要.docx
{案件名}_{YYYYMMDDHHmmss}-说明书.docx
```

案件名取自交底书中的「案件名称」字段，去掉占位符。
