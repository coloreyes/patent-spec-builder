# 专利说明书生成（主流程）

## 用途

将技术交底书或技术论文转换为标准**专利说明书**格式，输出两份文件：
1. **说明书摘要** — 一段式，200-400字
2. **说明书** — 完整格式，包含技术领域、背景技术、发明内容、附图说明、具体实施方式

## 输出格式要求

### 文件一：说明书摘要

**格式**：单一段落，不分节标题，约200-400字。

**内容结构**：
- 本发明公开了[方法/系统名称]，属于[技术领域]
- 针对现有[某技术]在[某场景]下面临的[问题1]以及[问题2]等缺陷
- 提出了[核心方案概述]
- 该方法首先[步骤1概述]，然后[步骤2概述]，进一步[步骤3概述]，最后[步骤4概述]
- 该方法在[某条件]下，能够[技术效果1]，提升[技术效果2]，具有[实际部署/应用价值]

### 文件二：说明书

**章节结构**（严格按以下顺序，无编号前缀）：

```
[发明名称]（首行，居中，黑体16pt加粗）

技术领域（黑体14pt加粗）
[1-2段]

背景技术（黑体14pt加粗）
[2-4段]

发明内容（黑体14pt加粗）
[目的段落]
[核心思路段落]
[步骤总述]
S1 [步骤名]：[详细描述]
S2 [步骤名]：[详细描述]
...
Sn [步骤名]：[详细描述]
[公式段落，OMath格式]
[系统模块段落]
[有益效果]（1）（2）（3）...

附图说明（黑体14pt加粗）
[图N为...说明]
[图片]
图N（居中，10.5pt）

具体实施方式（黑体14pt加粗）
[开头语段]
[术语解释段落（每个术语独立成段，含详细定义和引用）]
实施例1
[方法实施例，含详细公式推导和参数]
实施例2
[系统实施例，模块对应关系]
[应用例（含实验数据对比、基线方法描述、数据集描述）]
[结语段]
```

### 引用与术语规范（核心）

**专利说明书不使用 `[1]`, `[2]` 等参考文献编号，不使用独立的参考文献章节。**

所有外部内容（术语、数据集、基线方法、学术概念）必须通过**内联解释 + 参考文献引用**紧跟在所解释的内容之后：

1. **英文术语格式**：中文名称后紧跟 `（English Term）`，如：
   - `多模态特征融合（Multimodal Feature Fusion）：指...`
   - `概念漂移（Concept Drift）：指...`

2. **术语解释格式**：在「具体实施方式」开头部分，每个术语独立成段：
   ```
   术语中文名（English Term）：通用定义。[可选：在本发明中，...][详细解释内容]
   其XXX可参考文献【完整作者. 标题. 出版信息, 年份, 卷(期): 页码.】
   ```
   - 术语数量根据实际正文内容确定，提取所有必要的专业术语
   - 每个术语后必须跟随一段**引用段落**，格式为：`其[理论基础/相关研究/应用]可参考文献【完整文献信息】`
   - 引用段落使用宋体10.5pt，首行缩进0.74cm
   - 文献格式：`作者. 标题. 期刊/会议名, 年份, 卷(期): 页码.` 或 `作者. 标题. 出版社, 年份.`

3. **数据集描述格式**：
   ```
   数据集名称：详细描述其内容、规模和来源。[详细解释]
   其数据集详见【完整作者. 标题. 期刊名, 年份, 卷: 页码.】
   ```
   - 数据集引用段落格式同上

4. **基线方法描述格式**：
   ```
   方法名：详细的原理解释，作为本发明的基线方法。[详细解释]
   其方法详见【完整作者. 标题. 会议/期刊名, 年份, 页码.】
   ```
   - 基线方法引用段落格式同上

### 撰写风格规范

1. **不使用**「一、二、三」或「第一章」等数字编号作为节标题
2. **节标题**仅用：`技术领域`、`背景技术`、`发明内容`、`附图说明`、`具体实施方式`
3. **步骤编号**使用 `S1`、`S2`...`Sn` 格式
4. **有益效果编号**使用 `（1）`、`（2）`... 格式
5. **公式**使用 Word 原生 OMath 格式（通过 `tools/omath_converter.py` 中的 `formula_to_omath(latex)` 函数转换后 XML 注入）
6. **表格**使用 Word 原生表格，标题用 Caption 样式
7. **术语**首次出现时给出中英文全称及简称，如「持续学习（Continual Learning, CL）」
8. **段落**首行缩进0.74cm，行距1.5倍，宋体12pt
9. **实施例1 展开**：每个步骤（S1-Sn）在实施例1中需展开为 3-5 段详细描述：
   - 第1段：该步骤的整体描述
   - 第2段：「具体的，设...」引入符号和公式推导
   - 第3-4段：公式的解释、参数说明、边界条件
10. **引用段落**：每个术语、数据集、基线方法后必须跟随引用段落，使用宋体10.5pt
11. **结语**以「本领域的普通技术人员将会意识到...」收尾

### Word 生成技术要求

- 使用 `python-docx` 生成
- 公式使用 **OMath XML 注入**（Word 原生公式格式）：使用 `tools/omath_converter.py` 中的 `formula_to_omath(latex)` 函数将 LaTeX 公式转为 OMath XML 元素，通过 `paragraph._p.append(omath_element)` 插入段落
- OMath 命名空间：`http://schemas.openxmlformats.org/officeDocument/2006/math`
- 公式编号写在公式右侧：`（1）`、`（2）`...
- 表格使用 `document.add_table()`，标题用 `paragraph.style = 'Caption'`
- 正文段落使用 `style='Normal'`，首行缩进 `0.74cm`，行距 1.5 倍
- 节标题使用黑体 14pt 加粗，正文使用宋体 12pt
- 引用段落使用宋体 10.5pt，首行缩进 0.74cm

## 转换流程

1. **Read** 交底书全文（.md 或 .docx）及原始论文（如可用）
2. **提取**所有专业术语、数据集、基线方法、评估指标及对应的参考文献
3. **分析**交底书结构，提取各章节内容
4. **重组**为说明书格式，所有外部内容用内联解释紧跟其后
5. **扩写**具体实施方式（补充详细公式推导、参数设置、术语解释及引用）
6. **生成**说明书摘要（单段）
7. **输出**两份 Word 文档

## 输出文件命名

```
{案件名}_{YYYYMMDDHHmmss}-说明书摘要.docx
{案件名}_{YYYYMMDDHHmmss}-说明书.docx
```

案件名取自交底书中的「案件名称」字段，去掉占位符。
